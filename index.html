Para que serve?

O Termux transforma o Android em uma pequena estação Linux. Ele permite trabalhar com arquivos, testar sites, rodar scripts, usar GitHub e preparar projetos para publicação online.

Fluxo ideal de uso

1. Criar pasta do projeto
2. Editar ou copiar arquivos
3. Testar em localhost
4. Publicar online

Arquivos e Downloads

Organize arquivos soltos por tipo, encontre documentos e mantenha o celular limpo.

Criar e mover

mkdir PDFs Imagens Videos
cp arquivo.pdf PDFs/
mv arquivo.pdf PDFs/
mv antigo.pdf novo.pdf

Apagar com cuidado

rm arquivo.pdf
rmdir pasta_vazia
rm -r pasta

O comando rm -r remove a pasta com tudo dentro.

GitHub completo no Termux

Para guardar projetos no GitHub, instale o GitHub CLI e configure seu Git:

pkg install git gh -y
git config --global user.name "Marcelo Brandao"
git config --global user.email "segurocom@gmail.com"

Se o Git bloquear a pasta Downloads com “dubious ownership”, libere a pasta do projeto:

git config --global --add safe.directory /storage/emulated/0/Download/Projetos_HTML/Biblioteca

Para enviar um projeto já criado:

git init
git add .
git commit -m "primeira versao"
git branch -M main
git remote add origin URL_DO_REPOSITORIO
git push -u origin main

No GitHub, use um Personal Access Token como senha. A senha normal do GitHub não funciona no Git via HTTPS.

ZIP para empacotar projetos

O pacote zip é necessário para compactar arquivos e enviar projetos:

pkg install zip unzip -y
zip projeto.zip index.html
ls -lh *.zip

Vercel pelo Termux

Depois que o projeto tiver um index.html, publique com:

pkg install nodejs -y
npm i -g vercel
vercel --version
vercel --prod

Na primeira publicação, confirme o time, nome do projeto, diretório ./ e as configurações padrão.

Dica final

Treine sempre dentro de uma pasta de testes. Depois que o comando funcionar, aplique nos arquivos importantes.
